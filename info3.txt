=== GITHUB ACTIONS: ПЕРЕМЕННЫЕ И КОНТЕКСТ ===

=== УРОВНИ ПЕРЕМЕННЫХ (env) ===

1. Системные (встроенные в GitHub Actions)
2. Пользовательские на уровне workflow (env: в корне)
3. Пользовательские на уровне job (env: внутри job)
4. Пользовательские на уровне step (env: внутри step)

Каждый следующий уровень переопределяет предыдущий


=== СПОСОБЫ ВЫВОДА ПЕРЕМЕННЫХ ===

1. Через $ (только внутри run:)
   run: echo "$MY_VAR"fsfaf

2. Через выражение ${{ env.MY_VAR }} (везде в YAML)
   run: echo "${{ env.MY_VAR }}"
f

=== СРАВНЕНИЕ ПЕРЕМЕННЫХ ===

Сравнивать можно ТОЛЬКО через контекст:
   if: ${{ env.MY_VAR == 'production' }}

НЕЛЬЗЯ:
   if: env.MY_VAR == 'production'   (нет ${{}})
   if: ${{ $myvar == 'production' }} (лишний $)


=== $GITHUB_ENV (передача между шагами) ===

- Файл, в который можно записать переменную
- Становится доступна во ВСЕХ следующих шагах этого же job
- НЕ работает в текущем шаге

Пример:
   run: echo "MY_VAR=hello" >> $GITHUB_ENV

Можно записать несколько:
   run: |
     echo "VAR1=value1" >> $GITHUB_ENV
     echo "VAR2=value2" >> $GITHUB_ENV


=== КОНТЕКСТЫ ===

| Контекст | Пример | Откуда |
| env | ${{ env.MY_VAR }} | из секции env: |
| vars | ${{ vars.MY_VAR }} | из UI (Variables) |
| secrets | ${{ secrets.MY_SECRET }} | из UI (Secrets) |
| github | ${{ github.ref }} | встроенные |
| needs | ${{ needs.job.outputs.x }} | вывод другого job |


=== ПРАВИЛО ЗАПОМИНАНИЯ ===

- В if: → только ${{ env.ИМЯ == 'значение' }}
- В run: → можно $ИМЯ или ${{ env.ИМЯ }}
- Переменная из $GITHUB_ENV → только со следующего шага
